# 05 · NPC 系统

> 本文定义 NPC 的**核心字段、态度行为、交互与自主行为规则**。具体 NPC（叫什么、在哪、说什么台词）留待各区域内容填充。
>
> NPC 的**位置**归《[06-房间系统](06-房间系统.md)》、**战斗数值换算 / 商品 / 可教武学**归各自系统——本文锁字段与行为规则，不写数值公式。

## 1. 概述

NPC 是玩家在江湖中交互的主要对象——拜师、接任务、买卖、切磋打斗，几乎都通过 NPC 完成。它是把门派、武学、任务、物品等系统**串起来的接口**。

设计取向：

- **能力挂载，不设类型**：一个 NPC 能做什么，由它绑定的「能力」决定，而非靠固定类型枚举。同一个 NPC 可以既是师父、又发任务、还能交易（见 §3）。
- **模板 + 实例**：NPC 定义是模板；房间挂「NPC ID + 数量」生成实例（见《06-房间系统》）。
- **态度决定反应**：NPC 面对玩家挑衅、切磋、袭击时的反应由「态度」统一驱动（见 §4），不为单个 NPC 写特例。

## 2. NPC 核心字段

### 2.1 身份与外观

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| NPC ID | 稳定唯一标识 | `daopoqiong` | 必填，全局唯一；玩家指令定位与程序引用共用，不用中文名做关联键 |
| 名称 | NPC 名字 | 刀破穹 | 必填，游戏内显示 |
| 称号 | 名字前缀头衔 | — | 非必填 |
| 性别 | 性别 | 男 / 女 | 非必填，省略默认为男/雄  动物自动转换为 雄雌 |
| 年龄 | NPC 年龄 | 非必填，默认 18 | 供 look、称谓、任务条件等系统调用 |
| 种族 | 生物类别 | 人类  / 野兽 | 默认「人类」；影响文案与量词 |
| 量词 | 描述用量词 | 位 / 头 / 只 | 默认「位」，如「一头野猪」 |
| 外观描述 | 文字外貌形象 | 一段描述文字 | 必填，MUD 门面，玩家「看」NPC 靠它 |

### 2.2 属性与战斗资源

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| 等级 | NPC 等级 | int，默认 1 | 与玩家等级字段含义一致；影响击杀经验与掉落 |
| 力道 | 原生 · 外功根基 | 非必填 | 默认 20 |
| 气劲 | 原生 · 内功根基 | 非必填 | 默认 20 |
| 根骨 | 原生 · 体魄根基 | 非必填 | 默认 20 |
| 身法 | 原生 · 身手根基 | 非必填 | 默认 20 |
| 武学 | 已会武学及等级 | 非必填 | 拳脚 / 剑法 / 内功 / 轻功等，条目见《[03-武学系统](03-武学系统.md)》 |
| 门派身份 | 所属门派、辈分、身份 | 少林 · 四代 · 弟子 | 非必填，见《[02-门派系统](02-门派系统.md)》 |

> 「力道 / 气劲 / 根骨 / 身法」是原生属性（四维），含义与玩家一致（见《[01-角色系统](01-角色系统.md)》），**只有需要战斗的 NPC 才填**；商人、铁匠等功能 NPC 可全部留空。

#### 衍生属性的覆写与修正

NPC 的衍生属性（外攻 / 内攻 / 外防 / 内防 / 气血上限 / 内力上限 / 命中 / 闪避 / 会心 / 御心 / 招架，共 11 项，见《[08-数值计算系统](08-数值计算系统.md)》）默认由原生属性推导，**每一项**都另有两组可选字段用于调参：

| 字段组 | 含义 | 效果 |
|--------|------|------|
| 覆写值 | 直接指定该项衍生属性 | 跳过公式，直接用填的值 |
| 修正值 + 修正百分比 | 在推导结果上加码 | 作为加法层进公式，跟随公式浮动 |

取值优先级与分工（详见《08-数值计算系统》§8）：

- **覆写 > 修正 > 推导**：同一项属性既填覆写又填修正时，覆写生效、修正忽略；不同属性可混用（如 Boss 锁气血、其余走修正）。
- **普通 NPC 什么都不填**（纯推导）；**精英怪填修正**（比同级怪强一截，且随公式调平衡自动浮动）；**Boss 填覆写**（逐项锁数，如阶段战需要气血正好 50000）。

### 2.3 行为开关

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| 态度 | 面对玩家的行为倾向 | 4 种取值，见 §4.2 | 默认「中立」 |
| 所属 | 阵营归属 | 门派 / 势力 / 中立 / 敌对 | 决定立场，见 §4.1 |
| 不可被攻击 | 是否禁止玩家攻击 | 0 / 1，默认 0 | 1 表示不可被攻击；用于保护功能、剧情 NPC |
| 能否说话 | 能否语言交互 | 0 / 1，默认 1 | 不能说话的 NPC（野兽等）被挑衅直接开打 |
| 不自动恢复 | 关闭自动疗伤等恢复行为 | 0 / 1，默认 0 | 特殊演出 / 木桩用 |
| 刷新时间 | 被击杀后的复活时间 | int 秒，非必填 | 不填则用系统统一默认复活时间 |
| 闲聊 | 平时闲聊概率 + 台词池 | 非必填 | 台词也可以是一个动作描述 |
| 战斗喊话 | 战斗中喊话概率 + 台词池 | 非必填 | — |

> 不设「类型」字段：NPC 的功能由其绑定的能力体现。
> 不设「位置」字段：NPC 出现在哪个房间由《06-房间系统》的常驻 NPC 挂载管理。

## 3. 能力模块（按需挂载）

NPC 在核心字段之外，可按需挂载下列能力。**谁绑了什么能力，就能做对应的事**；能力的具体内容与数值归各自系统，NPC 这里只做关联。

| 能力 | 让 NPC 能做 | 挂载内容 | 归属系统 |
|------|-------------|----------|----------|
| 教学 | 当师父收徒、教武学 | 可教武学、可教等级上限 | 武学系统 |
| 交易 | 当商人买卖 | 售卖 / 收购列表 | 物品 / 货币 |
| 任务 | 发布 / 交付任务 | 关联任务 | 任务系统 |
| 战斗 | 可被挑战 / 击杀 | 击杀奖励（§8）、自动招式（§7） | 战斗系统 |

> 例：门派师父 = 核心字段 +「教学」+「任务」；铁匠 = 核心字段 +「交易」；山匪 = 核心字段 +「战斗」。典型组合见 §9。

## 4. 阵营与态度

### 4.1 所属（阵营）

| 取值 | 含义 |
|------|------|
| 门派 | 隶属六大门派之一（如师父、戒律） |
| 势力 | 隶属江湖势力（非可拜入组织） |
| 中立 | 不站队（多为功能 NPC：商人、铁匠） |
| 敌对 | 与玩家阵营对立（反派、匪类） |

### 4.2 态度

态度共 4 种，决定 NPC 面对玩家挑衅、切磋、袭击时的反应：

| 取值 | 名称 | 行为特征 | 适用 |
|------|------|----------|------|
| `peaceful` | 中立 | 默认。不主动攻击；健康时(气血高于 75%)接受切磋 | 普通村民、门派弟子 |
| `friendly` | 友善 | 不主动攻击；婉拒切磋；被下杀手仍会反击；护援玩家（§4.4） | 任务、商店、剧情 NPC |
| `aggressive` | 敌对 | 唯一见到玩家就主动攻击的类型 | 野怪、巡逻敌人、匪类 |
| `killer` | 凶恶 | 不主动开战，但被挑衅后极易下死手 | 杀手、Boss |

> 「所属」与「态度」相互独立：敌对阵营的 NPC 不一定主动攻击（可配 `killer`），中立 NPC 也可能因剧情转为敌对。
> 友善与中立 NPC 的默认交互都是完整的。是否能交易、教学、接任务，取决于挂载了哪些能力，不因态度默认减少交互。

### 4.3 切磋与挑衅规则

- 气血不足 75% 时，NPC 一般以「今天有些疲惫」婉拒切磋。
- 气血低于一半后，**任何态度**的 NPC 被袭击都会直接下死手反击。

### 4.4 友善 NPC 护援规则

友善 NPC 比中立 NPC 多一条默认行为：**护援玩家**。

当玩家在同一房间内被 `aggressive` NPC 主动攻击时，房间内态度为 `friendly` 的 NPC 可以主动协助玩家，对攻击者进入战斗。

护援触发需满足以下条件：

- 攻击玩家的一方必须是**态度为 `aggressive`** 的 NPC。
- 玩家必须是被动遭到攻击，而不是主动挑起战斗。
- 友善 NPC 必须具备战斗能力；没有战斗能力的友善 NPC 不参与战斗。
- 如果玩家主动攻击任何 NPC，其他友善 NPC 都不会帮助玩家，除非任务、剧情或特殊规则明确指定。

| 场景 | 友善 NPC 是否护援 |
|------|:----------------:|
| 敌对 NPC 主动攻击玩家 | ✅ 会护援（需具备战斗能力） |
| 玩家主动攻击敌对 NPC | ❌ 默认不护援 |
| 玩家主动攻击中立 NPC，中立 NPC 反击 | ❌ 不护援 |
| 玩家主动攻击友善 NPC，友善 NPC 反击 | ❌ 不护援 |

## 5. 交互

### 5.1 问答（ask）

- 玩家 `ask` NPC 某话题，NPC 回答对应内容；每个 NPC 配一张**话题表**（话题 → 回答）。
- 话题可以直接触发功能：发放任务、进入剧情流程等。
- 未专门设定的通用话题（姓名、本地风土、江湖消息）有默认问话演出，不至于冷场。

### 5.2 台词礼仪

NPC 台词根据双方身份自动使用合适称谓：尊称（阁下、少侠）、自谦（在下、老朽）、蔑称（小贼）等，符合江湖语境。称谓规则全局统一，不逐个 NPC 配置。

> 人物面板气泡台词（客户端查看 NPC 时的随机气泡）属客户端表现层，规则待移动端界面设计定，字段预留在话题表同层。

## 6. 自主行为

NPC 平时会自主进行以下活动，让世界「活」起来：

| 行为 | 说明 |
|------|------|
| 自动疗伤 | 受伤且身怀内功时运功疗伤、运功恢复，内力不足时打坐 |
| 闲聊 | 按设定概率随机说台词；台词也可以是一个动作（如巡逻走动） |
| 战斗喊话 | 战斗中按概率喊出战斗台词 |
| 随机巡逻 | 可设定为在相邻房间之间随机走动 |
| 携带物品 | 可携带并装备武器、衣物等 |

## 7. 自动招式（Boss 放招）

Boss 和精英 NPC 可配置「自动放招」规则，实现阶段性技能、召唤、喊话等演出。普通 NPC 不需要配置。

### 7.1 总体设定

| 项 | 说明 |
|------|------|
| 公共冷却 | 两次放招之间的最小间隔（秒） |
| 轮询间隔 | 战斗中轮询放招的间隔（秒），默认 4 秒 |
| 招式规则列表 | 若干条规则，每条见 §7.3 |

### 7.2 触发时机

| 时机 | 说明 |
|------|------|
| 普攻之后 | 每次普攻命中 / 落空后有机会接招式 |
| 战斗轮询 | 战斗中每隔几秒尝试放招 |
| 低血量 | 自身血量低于 30% 时的专属时机 |
| 血量阈值 | 血量跌破 90% / 80% / … / 10% 时触发，整场战斗只触发一次（适合 Boss 阶段转换） |

### 7.3 单条规则可设定的条件与效果

| 项 | 说明 |
|------|------|
| 动作类型 | 放招式 / 喊话 / 召唤等自定义行为 / 多步骤连招 |
| 目标 | 当前对手 / 自己 / 无目标 |
| 自身血量区间 | 只在自己血量处于某区间时触发 |
| 目标血量区间 | 只在对手血量处于某区间时触发（如斩杀技） |
| 内力消耗 | 内力不足则不放 |
| 触发概率 | 0~100% |
| 优先级与权重 | 多条规则同时满足时，先比优先级，再按权重随机 |
| 独立冷却 | 该招式自己的冷却时间 |
| 仅一次 | 整场战斗只触发一次 |
| 延迟释放 | 先亮起手势 / 台词，几秒后才真正出招，给玩家反应时间 |
| 释放台词 | 出招前后的场面描述文本 |

### 7.4 示例：黑风寨 Boss 熊魁

- 血量跌破 **70%**：大喊召唤两个土匪加入战斗；
- 血量跌破 **30%**：怒吼后对自己释放「血煞功」进入狂暴状态。

## 8. 击杀奖励

挂载「战斗」能力的 NPC 可配置被击杀后的奖励：

| 类别 | 说明 |
|------|------|
| 经验 | 可配置固定经验；未配置时按玩家等级取默认经验，受等级差修正 |
| 货币 | 如银两 |
| 物品掉落 | 每件物品可设定数量与掉落概率；另叠加该等级段的通用掉落池 |

> 组队分摊、怪物类型倍率、副本进度上报等规则依赖组队 / 副本系统，归远期（见 §11）。

## 9. 典型 NPC 组合示例

以下是常见 NPC 的**配置组合**，不是类型字段——都是「核心字段 + 态度 + 能力」拼出来的：

| 类型 | 典型设定组合 |
|------|--------------|
| 村民 / 剧情 NPC | `friendly` + 问答话题 + 不可被攻击 |
| 任务 NPC | `friendly` + 问答触发任务 + 任务能力 |
| 门派师父 / 弟子 | `peaceful` + 门派身份 + 武学 + 教学能力 + 可切磋 |
| 商人 / 铁匠 | `peaceful` 或 `friendly` + 交易能力 + 不可被攻击 |
| 野怪 | `aggressive` + 战斗能力 + 击杀奖励 |
| Boss | `killer` + 高战斗资源 + 自动招式（阶段技能 / 召唤） + 击杀奖励 |

## 10. 和其他系统的关系

| 系统 | 关系 |
|------|------|
| 房间系统 | 房间挂「NPC ID + 数量」决定 NPC 出现在哪里、刷多少个；NPC 自身字段归本文 |
| 区域系统 | 区域是 NPC 刷新和分布统计边界；NPC 数据按区域拆文件（§12） |
| 门派系统 | 师父 / 戒律等门派 NPC 挂门派身份与教学能力 |
| 武学系统 | NPC 的武学条目与可教武学引用武学定义 |
| 数值计算系统 | NPC 衍生属性的推导、覆写与修正的取值规则（§2.2、08 §8） |
| 战斗系统 | 切磋 / 袭击流程、击杀结算归战斗系统；四维 → 战斗数值的换算见《[08-数值计算系统](08-数值计算系统.md)》 |
| 任务系统 | 任务 NPC 通过问答话题发布 / 交付任务 |

## 11. 首版范围

首版只落以下内容：

- 核心字段（§2）与能力挂载框架（§3）。
- 阵营、4 种态度与护援规则（§4）。
- 问答交互（§5.1）。
- 闲聊一项自主行为；其余自主行为（自动疗伤、巡逻等）随对应系统接入。

暂不纳入（远期）：

- 善恶值、声望——玩家角色尚无对应字段，两边一起设计。
- 世仇标记（带仇怨的玩家出现时 NPC 主动攻击）——依赖仇怨系统。
- 自动招式（§7）——随《战斗系统》落地，本文先锁规则形态。
- 击杀奖励的组队分摊、副本进度上报——依赖组队 / 副本系统。
- 客户端头像、人物面板气泡——归移动端界面设计。

## 12. 数据落地

程序读取 TOML，不新增 CSV。

- NPC 数据**按区域拆文件**：`data/npcs/<区域ID>.toml`，与房间数据同一原则（见《[07-区域系统](07-区域系统.md)》§8）。
- 房间只挂 NPC ID，不重复保存 NPC 字段。
- 内容加载时校验：房间常驻 NPC 引用的 ID 必须已定义，悬空报错（与悬空出口同级错误）。

```toml
# data/npcs/xincun.toml
[[npc]]
id = "daopoqiong"
name = "刀破穹"
gender = "男"
age = 52
long = "一位鬓角斑白的老者，目光沉稳，透着久经风霜的干练。"
attitude = "friendly"
no_fight = true
# 能力挂载（任务 / 教学 / 交易）的字段结构待各系统定稿后补充

# 战斗 NPC 的衍生属性调参（§2.2，每项衍生属性同理，全部非必填）：
# max_qi = 50000            # 覆写：直接锁数，跳过公式（Boss 用）
# wai_at_add = 30           # 修正固定值：加到推导结果上（精英怪用）
# wai_at_add_ra = 20        # 修正百分比：+20%，与修正固定值配套
```
