# 05 · NPC 系统

> 本文定义 NPC 的**核心字段框架**。具体 NPC（叫什么、绑定哪些能力、说什么台词）留待《地图系统》《任务系统》等填充。
>
> NPC 的**位置**归《地图系统》、**对话台词**归后续《对话系统》、**战斗数值 / 商品 / 可教武学**归各自系统——本文只锁基础字段。

## 1. 概述

NPC 是玩家在江湖中交互的主要对象——拜师、接任务、买卖、切磋打斗，几乎都通过 NPC 完成。它是把门派、武学、任务、物品等系统**串起来的接口**。

设计取向：**一个 NPC 能做什么，由它绑定的「能力」决定，而非靠固定类型分类**。同一个 NPC 可以既是师父、又发任务、还能交易——按需挂载能力即可（见 §3）。

## 2. NPC 核心字段

每个 NPC 都具备以下基础字段：

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| NPC 编号 | 唯一标识 | 如 N1001 | 全局唯一，关联用 |
| 名称 | NPC 名字 | 刀破穹 | 游戏内显示 |
| 称号 | 头衔 | 村长 | 非必填 |
| 性别 | 性别 | 男 / 女 / 无 | 守卫、动物等可为「无」 |
| 年龄 | NPC 年龄 | 非必填，默认 18 | 基础设定值，可供 look、称谓、任务条件等系统调用 |
| 等级 | NPC 等级 | int，默认 1 | 与玩家等级字段含义一致，供战斗、任务、look 等系统调用 |
| 外观描述 | 文字外貌形象 | 一段描述文字 | MUD 门面，玩家「看」NPC 靠它 |
| 力道 | 四维 · 外功根基 | 非必填，战斗 NPC 用 | 影响外攻攻击、外攻防御 |
| 气劲 | 四维 · 内功根基 | 非必填，战斗 NPC 用 | 影响内攻、内防、内力 |
| 根骨 | 四维 · 体魄根基 | 非必填，战斗 NPC 用 | 影响气血、双防、韧性 |
| 身法 | 四维 · 身手根基 | 非必填，战斗 NPC 用 | 影响命中、闪避、会心、招架 |
| 所属 | 阵营归属 | 门派 / 势力 / 中立 / 敌对 | 决定立场 |
| 态度 | 对玩家的默认态度 | 友好 / 中立 / 敌对 | 决定默认交互与护援规则 |
| 不可被攻击 | 是否禁止玩家攻击 | 0 / 1，默认 0 | 1 表示不可被攻击；用于保护功能、剧情、核心 NPC |
| 刷新时间 | 被击杀后的复活时间 | int，单位秒，非必填 | 填写则按该秒数复活；不填则使用系统统一默认复活时间 |

> 「年龄」首版不直接参与战斗数值计算，但可作为文本表现和系统条件的基础设定值。
> 「等级」与玩家等级字段保持一致；未填写时默认 `1`。
> 「力道 / 气劲 / 根骨 / 身法」是 NPC 的四维，含义与玩家角色四维一致（见《[01-角色系统](01-角色系统.md)》），**只有需要战斗的 NPC 才填**；商人、铁匠等功能 NPC 可全部留空。四维的具体数值与战斗换算归《战斗系统》，本文只登记字段。
> 「不可被攻击」是逻辑开关：`1` 为不可被玩家主动攻击，`0` 为可按战斗规则被攻击。
> 「刷新时间」仅在 NPC 被击杀后生效。若未填写，则按系统统一默认复活时间处理（例如 180 秒，具体默认值归战斗 / 地图系统统一配置）。
> 不设「类型」字段：NPC 的功能由其绑定的能力体现，不用枚举硬分类。
> 不设「位置」字段：NPC 在哪张图由《地图系统》管理。

## 3. 能力模块（按需挂载）

NPC 在核心字段之外，可按需挂载下列能力。**谁绑了什么能力，就能做对应的事**；能力的具体内容与数值归各自系统，NPC 这里只做关联。

| 能力 | 让 NPC 能做 | 挂载内容 | 归属系统 |
|------|-------------|----------|----------|
| 教学 | 当师父收徒、教武学 | 可教武学、可教等级上限 | 武学系统 |
| 交易 | 当商人买卖 | 售卖 / 收购列表 | 物品 / 货币 |
| 任务 | 发布 / 交付任务 | 关联任务 | 任务系统 |
| 战斗 | 可被挑战 / 击杀 | 气血、武学、击败处理、掉落规则 | 战斗系统 |

> 例：门派师父 = 核心字段 +「教学」+「任务」；铁匠 = 核心字段 +「交易」；山匪 = 核心字段 +「战斗」。

## 4. 字段取值说明

### 所属（阵营）

| 取值 | 含义 |
|------|------|
| 门派 | 隶属六大门派之一（如师父、戒律） |
| 势力 | 隶属江湖势力（非可拜入组织） |
| 中立 | 不站队（多为功能 NPC：商人、铁匠） |
| 敌对 | 与玩家阵营对立（反派、匪类） |

### 态度

| 取值 | 含义 |
|------|------|
| 友好 | 可正常交互；在满足护援规则时，会主动帮助玩家 |
| 中立 | 可正常交互；不主动帮助玩家，也不主动攻击玩家 |
| 敌对 | 与玩家默认对立；关闭正常交互，按战斗规则攻击玩家或被玩家攻击 |

> 「所属」与「态度」相互独立：敌对阵营的 NPC 不一定主动攻击，中立 NPC 也可能因剧情转为敌对。
> 友好与中立 NPC 的默认交互都是完整的。是否能交易、教学、接任务，取决于 NPC 挂载了哪些能力，以及玩家是否满足对应条件；不因为「中立」默认减少交互。

#### 友好 NPC 护援规则

友好 NPC 比中立 NPC 多一条默认行为：**护援玩家**。

当玩家在同一房间内被「默认态度为敌对」的 NPC 主动攻击时，房间内默认态度为友好的 NPC 可以主动协助玩家，对攻击者进入战斗。

护援触发需满足以下条件：

- 攻击玩家的一方必须是**默认态度为敌对**的 NPC。
- 玩家必须是被动遭到攻击，而不是主动挑起战斗。
- 友好 NPC 必须具备战斗能力；没有战斗能力的友好 NPC 不参与战斗。
- 如果玩家主动攻击友好 / 中立 NPC，其他友好 NPC 不会帮助玩家。
- 如果玩家主动攻击敌对 NPC，友好 NPC 初版默认不自动帮助，除非任务、剧情或特殊规则明确指定。

示例：

| 场景 | 友好 NPC 是否护援 |
|------|:----------------:|
| 敌对 NPC 主动攻击玩家 | ✅ 会护援（需具备战斗能力） |
| 玩家主动攻击敌对 NPC | ❌ 默认不护援 |
| 玩家主动攻击中立 NPC，中立 NPC 反击 | ❌ 不护援 |
| 玩家主动攻击友好 NPC，友好 NPC 反击 | ❌ 不护援 |
