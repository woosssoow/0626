# 05 · NPC 系统

> 本文定义 NPC 的**核心字段框架**。具体 NPC（叫什么、绑定哪些能力、说什么台词）留待《地图系统》《任务系统》等填充。
>
> NPC 的**位置**归《地图系统》、**对话台词**归后续《对话系统》、**战斗数值 / 商品 / 可教武学**归各自系统——本文只锁基础字段。

## 1. 概述

NPC 是玩家在江湖中交互的主要对象——拜师、接任务、买卖、切磋打斗，几乎都通过 NPC 完成。它是把门派、武学、任务、物品等系统**串起来的接口**。

设计取向：**一个 NPC 能做什么，由它绑定的「能力」决定，而非靠固定类型分类**。同一个 NPC 可以既是师父、又发任务、还能交易——按需挂载能力即可（见 §3）。

## 2. NPC 核心字段

每个 NPC 都具备以下 7 个基础字段：

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| NPC 编号 | 唯一标识 | 如 N1001 | 全局唯一，关联用 |
| 名称 | NPC 名字 | 刀破穹 | 游戏内显示 |
| 称号 | 头衔 | 铁火堂主（可空） | 非必填 |
| 性别 | 性别 | 男 / 女 / 无 | 守卫、动物等可为「无」 |
| 所属 | 阵营归属 | 门派 / 势力 / 中立 / 敌对 | 决定立场 |
| 态度 | 对玩家的态度 | 友好 / 中立 / 主动敌对 | 「主动敌对」会主动攻击玩家 |
| 外观描述 | 文字外观 | 一段描述文字 | MUD 门面，玩家「看」NPC 靠它 |

> 不设「类型」字段：NPC 的功能由其绑定的能力体现，不用枚举硬分类。
> 不设「位置」字段：NPC 在哪张图由《地图系统》管理。

## 3. 能力模块（按需挂载）

NPC 在 7 个核心字段之外，可按需挂载下列能力。**谁绑了什么能力，就能做对应的事**；能力的具体内容与数值归各自系统，NPC 这里只做关联。

| 能力 | 让 NPC 能做 | 挂载内容 | 归属系统 |
|------|-------------|----------|----------|
| 教学 | 当师父收徒、教武学 | 可教武学、可教等级上限 | 武学系统 |
| 交易 | 当商人买卖 | 售卖 / 收购列表 | 物品 / 货币 |
| 任务 | 发布 / 交付任务 | 关联任务 | 任务系统 |
| 战斗 | 可被挑战 / 击杀 | 等级、气血、武学、刷新规则 | 战斗系统 |

> 例：门派师父 = 核心字段 +「教学」+「任务」；铁匠 = 核心字段 +「交易」；山匪 = 核心字段 +「战斗」。

## 4. 字段取值说明

### 所属（阵营）

| 取值 | 含义 |
|------|------|
| 门派 | 隶属六大门派之一（如师父、戒律） |
| 势力 | 隶属江湖势力（非可拜入组织） |
| 中立 | 不站队（多为功能 NPC：商人、铁匠） |
| 敌对 | 与玩家阵营对立（反派、匪类） |

### 态度

| 取值 | 含义 |
|------|------|
| 友好 | 可正常交互，不会攻击玩家 |
| 中立 | 默认不攻击，被攻击才还手 |
| 主动敌对 | 玩家靠近 / 进入范围会主动攻击 |

> 「所属」与「态度」相互独立：敌对阵营的 NPC 不一定主动攻击，中立 NPC 也可能因剧情转为敌对。
