## 0. 一句话

**Rust 驱动 + Lua 脚本热更**的现代化文字 MUD：网关与游戏服分进程，NATS 当总线，PostgreSQL + Redis 双持久层，数值走 TOML + Lua 双热更。性能、稳定优先，对 AI 友好。

## 1. 选型总表（定稿）

| 维度 | 选型 | 定位 |
|------|------|------|
| 驱动语言 | **Rust**（`tokio` 异步） | 内存安全 + 无 GC 停顿 + 海量长连接 |
| 脚本/热更 | **Lua**（`mlua`，lua54/vendored） | 命令、公式、行为，文件改了即热更 |
| 拓扑 | **网关 Gateway + 游戏服 Game Server**，两个独立进程 | 连接与逻辑解耦，游戏服可独立重启/热更 |
| 服务间总线 | **NATS** | pub/sub + request/reply；白送跨服广播/全局聊天/多游戏服路由 |
| 权威存储 | **PostgreSQL**（`sqlx`） | 账号、角色、持久数据；账号表/角色表分离 |
| 热/共享层 | **Redis** | 在线态、会话、缓存、排行榜、分布式锁、跨服热数据 |
| 数值-声明式 | **TOML + serde** | 四维/天赋/门派/校验规则；强类型校验当安全网 |
| 数值-公式 | **Lua** | 战斗、成长曲线、条件规则；进热更路径 |
| 客户端协议 | **telnet + WebSocket** | 网关接入，归一化为内部消息 |
| 内部消息序列化 | **MessagePack**（`rmp-serde`） | NATS payload；由 `protocol` crate 定义 |
| 密码 | **argon2** | 账号密码哈希，禁明文 |
| 日志/错误 | `tracing` / `anyhow`+`thiserror` | |

## 2. 拓扑

```
        客户端 (telnet / WebSocket)
                  │
        ┌─────────▼──────────┐
        │   网关 Gateway (Rust/tokio)         │  连接·协议编解码·登录鉴权·会话·路由
        └─────────┬──────────┘  ← 有连接态, 无游戏逻辑; 游戏服重启玩家不掉线
                  │  NATS (rmp-serde 消息, protocol crate 定义)
        ┌─────────▼──────────┐
        │  游戏服 Game Server (Rust + mlua)    │  世界态(内存权威)·game tick·Lua 热更逻辑/公式
        └───┬─────────────┬──┘
            │ 异步落库/读档     │ 热/共享态
   ┌────────▼─────┐  ┌───▼──────────┐
   │  PostgreSQL    │  │     Redis        │
   │ 账号·角色·持久  │  │ 在线态·会话·缓存· │
   │ (sqlx)         │  │ 排行·锁·跨服热数据 │
   └──────────────┘  └─────────────────┘
```

**核心原则（LPMud 思想的分布式版）：状态住驱动（Rust/内存），行为住脚本（Lua）。** 热更换逻辑不丢世界态——这是热更能安全做的根基。

## 3. NATS 主题约定（草案，实现时细化）

| 主题 | 方向 | 说明 |
|------|------|------|
| `game.<shard>.cmd` | 网关 → 游戏服 | 玩家指令；游戏服按 shard 用 queue group 消费 |
| `client.out.<session>` | 游戏服 → 网关 | 玩家专属输出；网关订阅 `client.out.>` 回推对应连接 |
| `world.broadcast.>` | 游戏服 → 网关 | 全局/区域广播、全服公告；网关扇出给所有连接 |
| `admin.reload` | 运维 → 游戏服 | 多游戏服时广播「重载脚本」信号 |

## 4. 持久化模型

- **权威在内存**：游戏服把世界态（实体/房间/在线角色）放内存里跑，保证 tick 性能。
- **异步落库（Write-Behind）**：定时 autosave + 关键事件触发 + 角色下线存档，批量写 PostgreSQL。
- **强一致路径**：账号注册、货币变动等（后续版本）走即时/事务写。
- **Redis 用途**：在线态 `online:<char_id>`、会话令牌、角色快照缓存（Cache-Aside，更新后删除）、排行榜（zset）、跨服**单写锁**（同一角色只在一个游戏服可写）。
- **缓存一致性**：先更新 PG 再删 Redis；删除失败重试。三大问题（穿透/击穿/雪崩）按需加空值缓存 / 互斥锁 / TTL 加随机。

## 5. 数值与配置（弃 CSV）

| 类型 | 格式 | 例子 | 理由 |
|------|------|------|------|
| 声明式数值 | **TOML**（serde 反序列化校验） | 四维基础、天赋预设、门派一览、字段校验规则 | 带注释、diff 友好、**改错了启动即报错** |
| 带逻辑/公式 | **Lua** | 战斗公式、成长曲线、条件规则 | 直接进热更路径，表达力满 |

> 原 `planning/data/*.csv` 的定稿数值，落地时转成 `data/*.toml`；带公式的部分迁到 Lua 脚本。

## 6. 热更与稳定性

- 状态在 Rust、逻辑在 Lua → 热更不丢档；`notify` 监听 `scripts/` 与 `data/`，改动即重载。
- 所有 Lua 调用包 `pcall`/`Result`：脚本报错只记日志、不崩服。
- `mlua` 设内存上限 + 指令计数钩子：掐死跑飞的死循环脚本。
- 重载进**全新沙箱环境**后原子替换，玩家无感。

## 7. 工程结构（Cargo workspace，落地参考）

```
crates/
  gateway/        # bin: 连接·鉴权·会话·路由(连 NATS + Redis)
  game-server/    # bin: 驱动核心 + mlua 宿主 + 世界(连 NATS + PG + Redis)
  protocol/       # lib: 网关↔游戏服消息定义(共享接缝, rmp-serde)
  persistence/    # lib: sqlx 数据访问层
scripts/          # Lua mudlib(热更)
data/             # TOML 数值表(热更)
migrations/       # sqlx 数据库迁移
deploy/           # docker-compose: nats + postgres + redis(本地一键起)
```

**主要依赖**：`tokio` · `mlua`(lua54,vendored) · `async-nats` · `sqlx`(postgres) · `redis`(deadpool) · `notify` · `toml`+`serde` · `rmp-serde` · `argon2` · `tokio-tungstenite` · `tracing` · `anyhow`/`thiserror`。

## 8. 与首版 MVP 的关系

首版功能边界**不变**（注册 → 建角色 → 进江湖 → 看信息，见 [01-角色系统](01-角色系统.md)）。本架构是承载它的地基：MVP 即跑通「网关 + 游戏服 + NATS + PG + Redis」全链路，业务上只做角色相关四件事。门派及之后仍按路线图后置（YAGNI）。
