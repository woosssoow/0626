# 08 · 数值计算系统

> 本文定义**四维 / 等级 / 武学 → 战斗衍生属性**的推导公式，是全游戏的数值中枢。移植自《侠客MUD》老服务端核心公式系统（xkformula），保留公式结构与系数，术语对齐本策划案。
>
> 衍生属性**在战斗中怎么用**（命中判定、伤害结算、会心触发）归《战斗系统》；武学的**系数数值**归《武学列表》——本文只管「属性怎么算出来」。

## 1. 概述

属性分两层，术语全案统一：

- **原生属性（输入层）**：不由公式算出、有独立成长途径的属性——四维（力道 / 气劲 / 根骨 / 身法，创建 / 配点）、人物等级（经验）、武学等级（潜能）。
- **衍生属性（输出层）**：纯由公式推导、自身无独立成长途径的属性——攻击、防御、气血、命中、闪避等 11 项（见 §5）。想提升它们只能去动输入：加四维、升级、练武学、穿装备。

由此有一条落库红线：**数据库只存原生属性，衍生属性永不落库**，随算随出（见 §9 属性快照）。本系统规定的就是这条「原生 → 衍生」推导链。

- **玩家和 NPC 共用同一套公式**：NPC 不填战斗资源时走推导（见《[05-NPC系统](05-NPC系统.md)》§2.2），保证玩家与 NPC 数值同源、强度可比。
- **一套骨架算所有**：11 项衍生属性共用一个公式骨架，只换参数（基础常数、驱动属性、成长率、武学系数），调平衡就是调参数表，不改公式结构。

## 2. 设计原则

- **加法分层，乘法收口**：各来源（武学核心 / 装备 / Buff / 称号 / NPC 修正）先各自算好再**相加**，最后一个「总百分比」乘区收口。层与层互不相乘，避免乘区叠爆。
- **来源归一化**：公式骨架**不逐个认识来源系统**——装备、Buff、称号、天赋、区域效果、NPC 修正等一切来源，统一归一化为按属性聚合的「修正集合」（固定值 / 百分比）再进公式。§4 点名的装备 / Buff / 称号只是当前已知来源的示例；新增来源（如天赋、套装、区域光环）只在自己系统内折算成标准修正投入，**不改公式骨架、不加新层**。
- **等级用平滑曲线**：等级收益走饱和曲线（越往后涨幅越小），全程无断点，不出现「差 1 级属性跳一截」。
- **参数外置**：所有系数进 TOML、公式逻辑走 Lua（见《[99-技术架构](99-技术架构.md)》），数值调整与公式改动都可热更，不动服务端代码。

## 3. 公式输入

| 输入 | 来源 | 说明 |
|------|------|------|
| 四维（力道 / 气劲 / 根骨 / 身法） | 《[01-角色系统](01-角色系统.md)》 | 每项衍生属性由指定的四维项驱动 |
| 人物等级 | 角色 / NPC 字段 | 进等级成长曲线 |
| 武学等级 | 《[03-武学系统](03-武学系统.md)》 | 当前激发武学的等级 |
| 武学公式系数 | 《武学列表》（待建） | 每门武学携带一张系数表，见 §3.1 |
| 门派加成 | 《[02-门派系统](02-门派系统.md)》 | 门派系数，作用于武学核心值；**首版全 0**，留作平衡钩子 |
| 装备 / Buff / 称号加成 | 各自系统（远期） | 加法层，系统未接入时为 0，见 §4 |

### 3.1 武学公式系数表

每门武学定义时携带一张**公式系数表**，声明它对各衍生属性的贡献强度：

`wai`（外攻） · `nei`（内攻） · `qi`（气血） · `neili`（内力） · `hit`（命中） · `dodge`（闪避） · `crit`（会心） · `non_crit`（御心） · `parry`（招架）

- 内功心法主要填 `qi / neili / non_crit`，套路（外功招式）主要填 `wai / nei / hit / crit / parry`，轻功主要填 `dodge / hit / crit / parry`——具体数值归《武学列表》。
- 未学任何对应武学时，回落到**基础武学**的系数表（见 §6）。

## 4. 统一公式骨架

11 项衍生属性都按同一骨架计算：

```
武学核心 = 基础常数 + 武学系数 × 武学等级 / 100 × (1 + 门派加成)

成长乘区 = 1 + 属性加成 + 核心成长率 + 等级成长率

最终值 = ( 武学核心 × 成长乘区
         + 装备加成 × (1 + 装备百分比)
         + Buff加成 × (1 + Buff百分比)
         + 称号加成 × (1 + 称号百分比)
         + NPC修正值 × (1 + NPC修正百分比)
         + 额外固定值
        ) × (1 + 总百分比)
```

各项含义：

| 项 | 说明 |
|------|------|
| 基础常数 | 该属性的裸值（没学武学、四维为 0 时的底），保证公式永不归零 |
| 武学系数 × 武学等级 / 100 | 武学是核心成长来源：等级 100 时恰好吃满一份系数 |
| 属性加成 | 驱动四维 × 对应系数（如外攻 = 力道 × 0.5%），见 §5 参数表 |
| 核心成长率 | 该属性的固定修正率，可为负（用于压闪避 / 会心的自然增长） |
| 等级成长率 | 人物等级饱和曲线，见下 |
| 装备 / Buff / 称号加成 | 各自独立的「固定值 × (1 + 自身百分比)」加法层，系统未接入时整层为 0 |
| NPC修正 | 仅 NPC 有的加法修正层，见 §7 |
| 额外固定值 | 个别属性的补充项（目前只有气血上限用到，= 等级 × 50） |
| 总百分比 | 最终乘区，所有「全局 +X%」类效果收口到这里 |

**等级成长曲线**（饱和曲线，早期涨得快、后期趋平，无断点）：

| 曲线 | 公式 | 上限（渐近值） | 适用 |
|------|------|:---:|------|
| 标准曲线 | `0.1 × 等级 / (等级 + 85)` | +10% | 除气血外的全部属性 |
| 气血曲线 | `0.4 × 等级 / (等级 + 15)` | +40% | 气血上限（另有 等级 × 50 固定值托底） |

## 5. 衍生属性参数表

| 衍生属性 | 基础常数 | 武学系数来源 | 武学等级取自 | 属性加成 | 核心成长率 |
|----------|:---:|--------------|:---:|------------------------|:---:|
| 外功攻击 | 20 | 当前套路 `wai` | 当前套路 | 力道 × 0.5% | +10% |
| 内功攻击 | 20 | 当前套路 `nei` | 当前套路 | 气劲 × 0.6% | +10% |
| 外功防御 | 20 | 内功 `wai` | 内功 | 力道 × 0.3% + 根骨 × 0.6% | +10% |
| 内功防御 | 20 | 内功 `nei` | 内功 | 根骨 × 0.6% + 气劲 × 0.3% | +10% |
| 气血上限 | 300 | 内功 `qi` | 内功 | 根骨 × 1.0% | +10% |
| 内力上限 | 100 | 内功 `neili` | 内功 | 气劲 × 1.6% | +80% |
| 命中 | 10 | 轻功 `hit` + 套路 `hit` | 轻功 | 身法 × 0.7% | +5% |
| 闪避 | 10 | 轻功 `dodge` | 轻功 | 身法 × 0.6% | −10% |
| 会心 | 10 | 轻功 `crit` + 套路 `crit` | 轻功 | 身法 × 0.5% | −5% |
| 御心 | 10 | 内功 `non_crit` | 内功 | 根骨 × 0.5% | +20% |
| 招架 | 10 | 轻功 `parry` + 套路 `parry` | 轻功 | 身法 × 0.6% | +5% |

设计意图备注：

- **闪避 / 会心的核心成长率为负**：压制这两项的自然增长，让它们主要靠武学和装备堆，避免人人天生高闪高暴。
- **内力上限核心成长率高达 +80%**：内力是技能消耗资源，基数需要足够宽裕。
- **命中 / 会心 / 招架吃「轻功 + 套路」双系数**，体现身手与招式并重；但**武学等级只取轻功等级**（老案如此，移植保留，是否改为各取各的等级见 §12 待复核）。
- 「御心」是会心的对抗属性（降低被会心概率），与「会心 vs 御心」「命中 vs 闪避」「招架」的对抗判定规则归《战斗系统》。

## 6. 武学来源选择规则

公式里的「当前套路 / 内功 / 轻功」按以下规则选取：

| 场景 | 取哪门武学 |
|------|-----------|
| 攻击类（外攻 / 内攻，及命中 / 会心 / 招架的套路份额） | **跟武器走**：当前装备武器对应的套路类型（剑法 / 刀法 / 棍法…）；空手时取拳脚 |
| 防御 / 气血 / 内力 / 御心 | 当前激发的**内功** |
| 闪避，及命中 / 会心 / 招架的轻功份额 | 当前激发的**轻功** |

**回落规则**：对应类别没有激发任何武学时，回落到该类别的**基础武学**（拳脚 / 基础内功 / 基础轻功，即《03-武学系统》品阶表中的「基础武学」），用其系数表兜底——保证新手号没拜师也能算出完整属性，公式链永不缺项。

## 7. NPC 取值规则

NPC 的衍生属性按三层优先级取值，与《05-NPC系统》§2.2「衍生属性的覆写与修正」同一套规则：

| 优先级 | 层 | 说明 |
|:---:|------|------|
| 1 | **覆写** | NPC 直接填了该项衍生属性（如气血上限），整条公式跳过，用填的值 |
| 2 | **修正** | NPC 配「修正固定值 + 修正百分比」，作为加法层进公式（§4 的「NPC修正」项），在推导基础上加码，随公式调参浮动 |
| 3 | **推导** | 什么都不填，按四维 / 等级 / 武学正常推导 |

- 优先级按**属性逐项判定**：同一项既填覆写又填修正时，**覆写生效、修正忽略**（不叠加）；不同属性可混用，如 Boss 锁气血、其余项走修正。
- 分工：普通 NPC 一律走推导；精英怪用修正在推导值上加码；Boss 才用覆写逐项锁数。

## 8. 属性快照

衍生属性**不逐回合现算**：进入战斗或属性来源变化时一次算全 11 项，存为「战斗属性快照」，战斗中直接读快照。

重算时机：装备变更、Buff 增减、武学升级 / 切换激发、人物升级。快照失效即重算，规则简单、无脏数据。

**公式 / 系数热更的失效规则**：快照携带公式版本号；TOML / Lua 热更通过校验后，在游戏 tick 边界**全局统一失效重算**——避免一部分实体持旧快照、一部分已重算，同一场对抗混用两代公式。战斗系统接入前，快照仅服务面板展示，同规则先行生效。

## 9. 和其他系统的关系

| 系统 | 关系 |
|------|------|
| 角色系统 | 四维是公式的驱动输入；四维的「影响/用途」列即本文参数表的落地 |
| 武学系统 | 每门武学携带公式系数表；武学等级是核心成长来源 |
| NPC 系统 | NPC 战斗资源未填时由本系统推导；三层取值规则见 §7 |
| 门派系统 | 门派加成系数作用于武学核心值，首版全 0 |
| 战斗系统 | 本系统产出衍生属性；命中判定、伤害结算、会心 / 御心 / 招架对抗规则归战斗系统 |
| 装备 / Buff / 称号 | 各占一个加法层，系统接入时填层即可，公式骨架不动 |

## 10. 首版范围

首版只落以下内容：

- 统一公式骨架（§4）与 11 项参数表（§5），生效项：**四维 + 等级 + 武学核心**。
- 武学来源选择与基础武学回落（§6）。
- NPC 三层取值（§7）。
- 属性快照（§8）。

暂不纳入：

- 装备 / Buff / 称号加法层——骨架保留、取值恒 0，随各系统接入。
- 门派加成——系数先全 0，门派平衡期启用。
- 管理员逐项覆写属性的调试通道——归运维工具，不进策划案。

## 11. 数据落地

程序读取 TOML，公式逻辑走 Lua（热更），不新增 CSV。

- 公式系数集中在 `data/formula.toml`：基础常数、属性系数、成长率、等级曲线参数。
- 每门武学的公式系数表随武学定义走（《武学列表》落地时定文件结构）。
- 内容加载时校验：11 项参数齐全、系数为数值、曲线参数非负，缺项报错。

```toml
# data/formula.toml（节选）
[level_curve]
standard = { factor = 0.1, half = 85 }   # 0.1 × L / (L + 85)
qi       = { factor = 0.4, half = 15 }   # 0.4 × L / (L + 15)

[derive.wai_at]        # 外功攻击
base = 20
attr = { str = 0.005 }
core_rate = 0.10
curve = "standard"

[derive.max_qi]        # 气血上限
base = 300
attr = { con = 0.01 }
core_rate = 0.10
curve = "qi"
extra_flat_per_level = 50
```
