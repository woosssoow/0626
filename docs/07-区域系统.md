# 07 · 区域系统

> 区域是一组房间的内容创作、广播和刷新边界。它不是独立空间单位，玩家实际位置仍然落在房间。

## 1. 设计定位

区域系统负责回答四个问题：

1. 房间属于哪一片地图。
2. 区域默认有哪些规则。
3. 区域级事件能影响哪些房间。
4. 内容制作时如何按区域组织房间、NPC、物品和刷新。

房间是玩家移动和交互的最小单位；区域是给策划、地图、刷新和广播使用的上层分组。例如「少林寺」「红枫镇」「新手村」都是区域，里面的山门、大殿、客栈、广场才是房间。

## 2. 设计原则

- **区域只做分组和默认规则**：不要把任务流程、战斗逻辑或副本阶段塞进区域。
- **房间规则优先于区域默认规则**：区域提供默认值，房间可以覆盖（见 §4）。
- **首版字段少**：先满足地图归属、区域默认限制和全局出生点。
- **区域ID稳定**：房间、NPC 分布、任务条件和日志都可能引用区域 ID，不用中文名做关联键。

## 3. 区域核心字段

| 字段 | 含义 | 示例 | 备注 |
|------|------|------|------|
| 区域ID | 稳定唯一标识 | `xincun` / `shaolin` | 必填，全局唯一，程序引用用它 |
| 区域名 | 玩家可见名称 | 新手村 / 少林寺 | 必填，用于地图和当前位置展示 |
| 区域类型 | 策划理解用分类 | 城镇 / 门派 / 野外 / 副本 | 非必填的自由标签，仅供策划分类与配置检查；程序规则一律看限制与脚本，不 switch 该字段 |
| 默认限制 | 区域级默认规则 | 禁战 / 禁快速移动 | 非必填，房间可覆盖（见 §4） |
| 出生房间 | 该区域默认入口 | `xincun_guangchang` | 非必填；新手村、门派入口等可用 |

区域同时天然承担两个**边界**职责，无需字段配置：

- **广播边界**：区域事件（如少林钟声）发给该区域内的在线玩家；跨区域广播走更高层频道。
- **刷新边界**：reset 只扫描本区域房间内的常驻 NPC / 物品，不跨区域扫描。

### 3.1 全局出生点

`areas.toml` 顶层定义**全局唯一**的出生点（区别于区域级「出生房间」）：

```toml
spawn = "xincun_guangchang"
```

- 新角色进入江湖的初始房间。
- 玩家存档位置（`location`，见《[01-角色系统](01-角色系统.md)》）指向的房间在内容更新后不存在时，**回退到全局出生点**。
- 内容加载时校验：全局出生点必须存在且指向有效房间，否则启动报错。

## 4. 默认限制与覆盖

区域默认限制用于减少重复配置。常用限制：

| 限制 | 含义 | 常见区域 |
|------|------|----------|
| 禁止战斗 | 不能主动发起战斗 | 城镇中心、门派大殿 |
| 禁止快速移动 | 不能车马、传送或快速离开 | 副本、监狱、剧情区域 |
| 禁止坐骑 | 不能骑乘进入 | 室内区域、寺庙、山洞 |
| 禁止武器 | 不能携带或装备武器进入 | 解剑岩、门派圣地 |

**覆盖规则**：同一限制如果区域和房间都配置，**以房间配置为准**；房间没有配置的项继承区域默认值。这允许「城镇禁战但擂台房间开战」而不必为擂台单划区域。

落地格式必须支持「显式关闭」，因此限制用**布尔表**而非列表（列表无法表达「房间显式豁免某项」）：

```toml
# areas.toml —— 区域默认
[[area]]
id = "hongfengzhen"
name = "红枫镇"
default_flags = { no_combat = true }

# rooms/hongfengzhen.toml —— 擂台房间显式覆盖
[[room]]
id = "hongfengzhen_leitai"
flags = { no_combat = false }   # 未提及的项仍继承区域默认
```

## 5. 区域事件

区域事件是区域内广播和触发的最小公共机制，首版只需要定义范围，不需要复杂事件系统。

| 事件 | 例子 | 处理边界 |
|------|------|----------|
| 区域广播 | 少林钟声传遍少林寺 | 发给该区域在线玩家 |
| 区域刷新 | 红枫镇每天刷新摊贩和杂物 | 只扫描该区域房间 |

「戒严时城门关闭」类**运行时改变区域规则**的玩法属远期：与房间/出口一样，区域配置也**永远无状态**（见《06-房间系统》§8.3），届时以「条件读取玩家/世界标记」实现，不修改区域配置态。

跨区域事件不归区域系统处理，例如全服公告、跨服活动和排行榜推送，后续走频道或服务端事件系统。

## 6. 和其他系统的关系

| 系统 | 关系 |
|------|------|
| 房间系统 | 每个房间通过 `area_id` 归属一个区域；玩家位置仍是 `room_id` |
| NPC 系统 | NPC 具体出生在房间；区域只作为刷新和分布统计边界 |
| 任务系统 | 可用区域作为条件范围，如「在少林寺内完成」 |
| 门派系统 | 门派驻地通常对应一个或多个区域 |
| 移动端界面 | 区域名用于地图分组、当前位置标题和导航层级 |

## 7. 首版范围

首版只落以下内容：

- 区域核心字段。
- 全局出生点。
- 房间引用 `area_id`。
- 区域默认限制与房间覆盖。

暂不纳入：

- 跨区域大地图寻路。
- 天气、昼夜、季节。
- 区域归属争夺。
- 帮派建设和地盘经营。
- 大规模动态事件。

## 8. 数据落地

程序读取 TOML，不新增 CSV。

- 区域与全局出生点定义在 `data/areas.toml`。
- 房间数据**按区域拆文件**：`data/rooms/<区域ID>.toml`，避免单文件膨胀与多人协作冲突。
- 房间的区域归属**由所在文件名推定**（`rooms/<区域ID>.toml` 里的房间就属于该区域），条目内不写 `area_id`，也不保存区域名。
- 内容加载时校验：房间文件名对应的区域ID必须在 `areas.toml` 里有定义，悬空报错（与悬空出口同级错误）。

```toml
# data/areas.toml
spawn = "xincun_guangchang"          # 全局出生点

[[area]]
id = "xincun"
name = "新手村"
kind = "城镇"
spawn_room = "xincun_guangchang"
default_flags = { no_combat = true }
```

```toml
# data/rooms/xincun.toml —— 文件名即区域ID，房间条目内不写 area_id
[[room]]
id = "xincun_guangchang"
name = "新手村·广场"
desc = "青石铺地的村中广场。"
exits = { north = "yuelai_kezhan" }
```
