# 09 · 物品系统

> 本文定义物品的**通用字段与能力块框架**——消耗品、装备、秘籍、材料等全部物品共用这一套定义。具体物品（叫什么、什么属性、哪里掉）留待各内容表填充。
>
> 物品的**存放规则**（容量 / 堆叠 / 溢出）归《[04-背包系统](04-背包系统.md)》、**货币**归《货币系统》（不是物品，不占格）、**场景可查看物件**（水井、铜钟）归《[06-房间系统](06-房间系统.md)》§5——本文只管「一件物品是什么」。

## 1. 概述

物品是玩家成长与经济流转的载体——打怪掉落、商店买卖、任务奖励、装备穿戴、吃药疗伤，都绕不开物品。它是把 NPC（交易 / 掉落）、房间（常驻物品）、角色（装备槽）、武学（秘籍）等系统**串起来的实体**。

设计取向与 NPC 系统同一范式：

- **能力挂载，不设类型**：一件物品能干什么，由它挂载的「能力块」决定，而非固定类型枚举。同一件物品可以既能装备、又能阅读（如一本可佩挂的兵法帛书），互不冲突（见 §3）。
- **模板 + 引用**：物品定义是模板；背包、房间、掉落表、商店只挂「物品 ID + 数量」，不重复保存物品字段。
- **「消耗品 / 装备 / 材料」只是组合叫法**：都是「通用字段 + 能力块」拼出来的典型组合（见 §4），不是字段。

## 2. 通用字段

所有物品都具备以下字段：

| 字段 | 含义 | 取值 / 示例 | 备注 |
|------|------|-------------|------|
| 物品 ID | 稳定唯一标识 | `jinchuang_yao` | 必填，全局唯一；程序引用与掉落 / 商店 / 任务关联共用 |
| 名称 | 物品名 | 金疮药 | 必填，游戏内显示 |
| 描述 | 文字描述 | 一段描述文字 | 必填，玩家「看」物品靠它 |
| 量词 | 描述用量词 | 瓶 / 把 / 本 / 块 | 默认「个」，如「一瓶金疮药」 |
| 品质 | 物品分档 | 白 / 绿 / 蓝 / 紫 / 金 | 默认「白」；决定显示颜色与价值感，见 §2.1 |
| 堆叠上限 | 一格可叠多少 | int，默认 1 | 1 = 不可堆叠（装备类必为 1）；上限与占格规则见《04-背包系统》§3 |
| 唯一 | 唯一装备标签 | 0 / 1，默认 0 | 1 = 每角色限装 1 件，规则见《[01-角色系统](01-角色系统.md)》§5.3 |
| 绑定 | 绑定方式 | `none` / `pickup` / `equip`，默认 `none` | 不绑定 / 拾取后绑定 / 装备后绑定，规则见 §2.2 |
| 价值 | 物品的内在价值 | int，非必填 | 不填 = 无价之物，不可出售给 NPC（任务物品等）；商店买卖价由价值 × 系数派生，系数归《商店系统》；货币单位归《货币系统》 |
| 不可丢弃 | 禁止丢弃 / 出售 | 0 / 1，默认 0 | 任务关键物品用，防误删 |

> **价值只是一个数，价格是商店行为**：NPC 回收价 = 价值 × 回收系数、商店售卖价 = 价值 × 售卖系数，系数是商店系统的全局参数（如回收 0.8）。调经济只动系数，不逐个改物品。

### 2.1 品质

品质是**分档标签**，管显示颜色与价值预期，不直接进战斗公式（装备属性写在装备块里，品质只约束其数值区间）：

| 品质 | 定位 |
|------|------|
| 白 | 普通：常规掉落、商店常货 |
| 绿 | 优良：小怪较好掉落 |
| 蓝 | 精良：精英怪、高级商店 |
| 紫 | 珍稀：Boss 掉落、困难任务 |
| 金 | 传说：远期，首版不发放 |

> 与武学品阶（基础 / 进阶 / 上乘 / 镇派）是两套体系，互不映射。各品质的数值区间归数值表。

### 2.2 绑定

绑定用于控制物品在玩家间的流转，三种取值：

| 取值 | 含义 | 适用 |
|------|------|------|
| `none` | 不绑定（默认） | 常规掉落、材料、商店货 |
| `pickup` | 拾取后绑定：到手即绑 | 任务奖励、Boss 专属掉落 |
| `equip` | 装备后绑定：穿戴过才绑 | 装备——可先流通，穿过即归属 |

**已绑定物品的通行规则**——绑定只封「玩家 → 玩家」这一条通道，其余照常：

| 通道 | 已绑定物品 |
|------|:---:|
| 交易给玩家（面对面 / 摆摊 / 拍卖等） | ❌ 禁止 |
| give 给 NPC（任务提交、剧情交付） | ✅ 允许 |
| 卖给 NPC 商店 | ✅ 允许（换下来的旧装备要有出口） |
| 丢弃 | ✅ 允许，但**直接消失**，见下 |

**丢弃规则**——是否落地看「落地有没有意义」：

- **已绑定物品**丢弃后**直接消失**，不落在房间（绑定跟人走，落地被拾取无意义）。
- **无价值物品**（未填价值）丢弃后同样**直接消失**（落地即房间垃圾）。
- 其余物品丢弃后落在当前房间，可被他人拾取；落地物品的宿主（内存地面容器）、清理计时与重启行为见《[10-运行时实体系统](10-运行时实体系统.md)》§6。
- 丢弃即消失属**永久销毁**，客户端需二次确认，防误删。

补充约束：

- 「装备后绑定」只对挂了可装备块的物品合法（消耗品配 `equip` 属配置错误，加载报错）。
- 「已绑定」是**物品实例**的状态：同种装备穿过的已绑、没穿过的未绑，随存档走。`equip` 只允许配在不可堆叠物品上，`pickup` 到手即绑、全部实例状态一致，不影响堆叠。
- 绑定与「不可丢弃」「无价值」是三个独立开关，正交组合使用。

## 3. 能力块（按需挂载）

物品在通用字段之外，可按需挂载下列能力块。**挂了什么块，就能干什么事**：

| 能力块 | 让物品能 | 挂载内容 | 关联系统 |
|--------|----------|----------|----------|
| 可使用 | 被「使用 / 服用」消耗 | 使用效果、单次消耗数量、使用条件 | 效果见 §3.1 |
| 可装备 | 穿戴到装备槽 | 目标槽位、武器类型、属性加成 | 槽位见《01-角色系统》§5，加成进《[08-数值计算系统](08-数值计算系统.md)》装备层 |
| 可阅读 | 被「研读」 | 关联武学、解锁 / 自修效果 | 秘籍三册见《[02-门派系统](02-门派系统.md)》§2.4，读书自修见《[03-武学系统](03-武学系统.md)》§7 |
| 扩容背包 | 装入背包槽提升容量 | 提供格数 | 物品态 / 装备态规则见《04-背包系统》§2.3 |

什么块都不挂的物品就是**纯材料 / 任务信物**：只能拾取、持有、交付、交易。

> 不设「物品类型」字段：是不是装备、消耗品，看挂了什么块，不另存一份类型信息（与 NPC 不设类型同一原则）。客户端背包分页（装备 / 消耗品 / 材料 / 任务）由能力块**推导**：挂可装备块→装备页、挂可使用块→消耗品页、不可丢弃且无价值→任务页、其余→材料页，推导规则属界面层。

### 3.1 可使用块

| 项 | 说明 |
|------|------|
| 使用效果 | 恢复气血 / 恢复内力 / 解毒 / 获得 Buff / 触发剧情标记等 |
| 消耗数量 | 单次使用消耗几个，默认 1 |
| 使用条件 | 等级 / 状态限制（如仅战斗外可用），非必填 |
| 使用冷却 | 同类使用间隔（秒），非必填 | 

> 效果的具体数值（回多少血）随物品定义走；Buff 本体归《战斗系统》，首版可使用块只做**即时恢复类**效果。

### 3.2 可装备块

| 项 | 说明 |
|------|------|
| 目标槽位 | 13 槽之一：武器（主 / 副）/ 护具 7 槽 / 饰品 4 槽，见《01-角色系统》§5.1 |
| 武器类型 | 仅武器填：主手 / 单手 / 双手 / 副手，占槽规则见《01-角色系统》§5.2 |
| 套路类型 | 仅武器填：剑法 / 刀法 / 棍法 / 拳脚…，决定公式取哪门套路（《08-数值计算系统》§6） |
| 属性加成 | 若干条「衍生属性 + 固定值 / 百分比」，进《08-数值计算系统》§4 的装备加法层 |
| 装备条件 | 等级 / 门派 / 性别等限制，非必填 |

> 属性加成只允许加**衍生属性**（外攻 / 命中 / 气血上限…），不允许直接加四维（原生属性只走角色成长），避免装备绕过成长体系。

## 4. 典型物品组合示例

以下是常见物品的**配置组合**，不是类型字段——都是「通用字段 + 能力块」拼出来的：

| 俗称 | 典型设定组合 |
|------|--------------|
| 消耗品（药品 / 食物） | 可堆叠 + 可使用（恢复效果） |
| 武器 | 不可堆叠 + 可装备（武器槽 + 武器类型 + 套路类型 + 属性加成） |
| 护具 / 饰品 | 不可堆叠 + 可装备（对应槽位 + 属性加成） |
| 秘籍 / 书籍 | 不可堆叠 + 可阅读（关联武学） |
| 扩容背包 | 不可堆叠 + 扩容背包块 |
| 材料 | 可堆叠，不挂块 |
| 任务信物 | 不挂块 + 不可丢弃 + 无价值 |

## 5. 和其他系统的关系

| 系统 | 关系 |
|------|------|
| 运行时实体系统 | 落地物品的内存宿主与清理计时、物品实例标记（已绑定）的持久化边界，见《[10-运行时实体系统](10-运行时实体系统.md)》§6 / §7 |
| 背包系统 | 背包管「放得下放不下」；本文提供每件物品的堆叠上限；扩容背包的装备态规则归背包系统 |
| 角色系统 | 装备槽位框架（13 槽、武器四类、唯一标签）在角色系统，可装备块按槽位填 |
| 数值计算系统 | 可装备块的属性加成进装备加法层；装备变更触发属性快照重算 |
| NPC 系统 | NPC 交易能力的售卖 / 收购列表、击杀奖励的掉落物，挂物品 ID |
| 房间系统 | 房间常驻物品挂「物品 ID + 数量」；可查看物件不是物品，归房间系统 |
| 门派 / 武学系统 | 秘籍 / 书籍是挂「可阅读块」的物品；解锁与自修规则归各自系统 |
| 货币系统 | 货币**不是物品**、不占格（《04-背包系统》§6）；价值的货币单位归货币系统 |
| 商店系统 | 买卖价由物品价值 × 系数派生（回收 / 售卖系数为商店全局参数）；售卖 / 收购列表挂在 NPC 交易能力上 |
| 任务系统 | 任务物品无特殊字段，靠「不可丢弃 + 无价值」组合实现 |

## 6. 首版范围

首版只落以下内容（**依赖《战斗系统》的能力先锁字段、后置生效**，见《[00-总览](00-总览.md)》§3.1）：

- 通用字段（§2）与品质分档（白～紫，金不发放）。
- 绑定字段与丢弃规则（§2.2）——字段与规则现在锁定；「禁止交易给玩家」随《交易系统》上线生效，丢弃消失规则首版即生效。
- 可使用块与可装备块的**字段框架**——可使用块的恢复效果作用于「当前气血 / 内力」，该资源归《战斗系统》，生效后置；可装备块随《战斗系统》接入。
- 典型组合（§4）中的材料、任务信物先行；消耗品、武器 / 护具随《战斗系统》进度接入。

暂不纳入（远期）：

- 耐久与修理。
- 套装效果。
- 强化 / 打造 / 洗练——届时立独立玩法文档，不塞进本文。
- 可阅读块的落地——随《武学列表》与自修数值定稿。
- 金品质发放。

## 7. 数据落地

程序读取 TOML，不新增 CSV。

- 物品数据**按类别拆文件**：`data/items/weapons.toml`、`data/items/consumables.toml`、`data/items/materials.toml` 等。物品无地理归属（通用药材全地图用），按类别拆比按区域拆更少重复；文件划分是**内容组织习惯**，程序不依赖文件名做逻辑。
- 背包、房间、掉落表、商店只存物品 ID，不重复保存物品字段。
- 内容加载时校验：所有被引用的物品 ID 必须已定义，悬空报错（与悬空出口、悬空 NPC 同级错误）；可装备块的槽位、武器类型必须是合法枚举值；`bind = "equip"` 仅限挂可装备块且不可堆叠的物品。

```toml
# data/items/consumables.toml
[[item]]
id = "jinchuang_yao"
name = "金疮药"
desc = "祖传金疮药，敷之可止血生肌。"
unit = "瓶"
quality = "白"
stack = 99
value = 50
use = { effect = "restore_qi", amount = 200 }

# data/items/weapons.toml
[[item]]
id = "qingfeng_jian"
name = "青锋剑"
desc = "一柄寒光凛凛的青锋长剑。"
unit = "把"
quality = "绿"
value = 800
bind = "equip"     # 装备后绑定；不填默认 none 不绑定
equip = { slot = "main_hand", weapon_type = "单手", skill_type = "剑法", wai_at = 15, hit_ra = 3 }
```
