# 00 · 总览

> 武侠 MUD 游戏的顶层设计。本文件只讲「是什么、做什么、按什么顺序做」，具体系统拆到各自的 `docs/NN-xxx.md`。

## 1. 游戏定位

- **题材**：武侠江湖，初入江湖的少侠从拜师、习武、闯荡到一方人物。
- **形态**：文字 MUD，多人在线，以指令 / 文本交互为核心。
- **基调**：低风险、可验证、易调参。首版不堆复杂随机，先把核心流程跑通。

## 2. 核心玩法循环（目标形态）

```
注册账号 → 创建角色 → 进入江湖
   → 拜入门派 → 修习武学 → 历练升级
   → 装备成长 → 完成任务 / 押镖 / 奇遇
   → 江湖社交 / 切磋 → 成为一方人物
```

> 首版只做到「注册 → 创建角色 → 进入江湖」并展示基础角色信息，**门派之后的统统暂不纳入**。

## 4. 首版 MVP 边界

**纳入**：账号注册、登录校验、角色创建（一账号一角色）、四维 / 天赋 / 福缘等基础数据、进入游戏、展示角色信息。

**暂不纳入**：门派、武学、装备、背包、货币、任务、战斗公式、社交、公会、排行榜、赛季继承。

## 5. 设计原则

- **KISS**：字段少、含义清楚、默认值明确。
- **YAGNI**：只实现当前明确需要的；不预留用不到的特性（如首版不做随机天赋、多角色、改名）。
- **可测可调**：首版用固定规则保证公平，参数集中、易于调整。
- **数据分层**：账号数据与角色数据分表保存——账号管登录安全，角色管游戏内身份与成长。

## 6. 技术取向（预案，未定）

- **后端**：Python（FastAPI / Django），账号表与角色表分离。
- **连接**：长连接（WebSocket / Telnet），契合 MUD 的实时文本交互。
- **数据**：策划数值以 `data/*.csv` 为准，程序直接读取。
