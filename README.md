# 武侠 MUD 游戏策划案

一款文字武侠 MUD（Multi-User Dungeon）游戏的设计文档仓库。

## 项目定位

- **类型**：武侠题材文字 MUD，多人在线
- **当前阶段**：策划设计阶段（暂无代码）
- **设计原则**：字段少、含义清楚、默认值明确、后续可扩展；首版避免复杂随机和不可验证规则（KISS + YAGNI）

## 仓库结构

```
mud/
├── README.md                # 本文件，项目总览
├── docs/                    # 策划文案（Markdown，人看的）
│   ├── 00-总览.md           # 世界观、核心玩法、版本路线图
│   ├── 01-角色系统.md       # 首版：角色创建与基础数据定义（MVP）
│   ├── 02-门派系统.md       # 六大门派、师门系统骨架（照 XK 老案）
│   └── 03-武学系统.md       # 武学通用规则：品阶/成长/学习/三上限
├── data/                    # 数值表（CSV，程序直接读）
│   ├── 天赋预设.csv
│   ├── 字段校验规则.csv
│   └── 门派一览.csv
└── design-source/           # Excel 源文件，仅作计算草稿，不当最终产物
    ├── MUD游戏策划案_角色定义首版.xlsx
    └── XK策划案.xlsx         # 《侠客MUD》老策划案，全系统参考蓝本
```

## 文档约定

- **文案 / 规则 / 世界观 / 剧情** → 写在 `docs/*.md`，方便 diff 和跳转。
- **纯数值表**（经验曲线、四维成长、掉落率）→ 在 Excel 里用公式算，**定稿后导出 CSV** 放进 `data/`，CSV 才是程序读取的准。
- **Excel 源文件** 只放在 `design-source/` 当计算草稿，**不作为最终交付物**。
