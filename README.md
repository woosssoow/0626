# 武侠 MUD 游戏策划案

一款文字武侠 MUD（Multi-User Dungeon）游戏的设计文档仓库。

## 项目定位

- **类型**：武侠题材文字 MUD，多人在线
- **当前阶段**：策划设计阶段（暂无代码）
- **设计原则**：字段少、含义清楚、默认值明确、后续可扩展；首版避免复杂随机和不可验证规则（KISS + YAGNI）

## 仓库结构

```
mud/
```

## 文档约定

- **文案 / 规则 / 世界观 / 剧情** → 写在 `docs/*.md`，方便 diff 和跳转。
- **纯数值表**（四维成长、天赋、门派表）→ 声明式数值以 **TOML** 为程序读取的准（serde 强校验），带公式逻辑走 **Lua**，二者皆热更，详见 [99-技术架构](docs/99-技术架构.md)。**不再用 CSV**（现存 `data/*.csv` 待迁移）。
- **Excel 源文件** 只放在 `design-source/` 当计算草稿，**不作为最终交付物**。
